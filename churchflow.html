Risco de Perda

Voluntários esquecem de copiar arquivos ou deletam acidentalmente. Perder fotos de um evento importante é irreversível.

Desorganização

Pastas com nomes errados, datas misturadas e cultos (Manhã/Noite) no mesmo diretório. Um caos para encontrar arquivos depois.

Trabalho Manual

Renomear 500 fotos manualmente ou criar pastas toda semana consome tempo precioso da equipe de mídia.

Workflow Inteligente

A complexidade do Python escondida atrás de uma interface simples.

Deep Scan

O sistema lê o EXIF das fotos e separa eventos automaticamente baseado no horário (ex: Culto Manhã vs Noite).

Visual Select

Interface de Cards. O voluntário vê o que já foi baixado (Verde) e o que é novo (Cinza). Zero confusão.

Auto Backup

Copia, renomeia a pasta no padrão correto (AAAA-MM-DD) e marca o cartão como "Concluído".

Roadmap de Desenvolvimento

Prioridades definidas para garantir segurança e performance.

SafeCopy (shutil)

High Priority

Implementação robusta de cópia de arquivos para garantir que nenhum byte seja corrompido durante o backup.

Turbo Thumbnails

High Priority

Sistema de cache assíncrono para evitar travamentos ao carregar pastas com centenas de fotos pesadas.

Modo Foco

Planned

Navegação otimizada por teclado (Setas) para agilizar o processo de curadoria e seleção.

Integrity Check

Backlog

Verificação via Hash (Checksum) para segurança extra contra corrupção silenciosa de arquivos.
